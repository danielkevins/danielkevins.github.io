World Layoffs SQL data cleaning and EDA

Introduction

This project contains Data Cleaning and EDA using SQL of the world layoffs dataset.

Problem

In Data Analysis, the analyst must ensure that the data is 'clean' before doing any analysis. 'Dirty' data can lead to unreliable, inaccurate and/or misleading results. Garbage in = garbage out.
Here are some steps to prepare the data set for analysis :
1. copy raw data.
2. Check for duplicate entries and remove them.
3. Remove extra spaces and/or other invalid characters.
4. Check for null or empty values.
5. Remove null value or blank values.

Data Set Used

The data used is a collection of layoff data from companies around the world. For the data cleaning and EDA process, I using:
1. Basic select statements (select, where, group by).
2. Aggregate functions (count, sum, min, max)
3. Joins
4. CTE's, temp tables and views
5. string & date manipulation functions.
6. Window functions (rank, row_number)
